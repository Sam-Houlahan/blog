HTML,CSS,DOM

Post by Sam Houlahan, February 27, 2017.

Can you think of an analogy to describe HTML, CSS and the DOM?
How would you describe it to your non-technical friend?

My analogy would be to think of your webpage as a tree that has branches and flowers. The tree itself being HTML which has a body, head and elements as the branches. This is the main structure of a web page, now on this tree it has lots of pretty colorful flowers this would be your CSS aka Cascading Style Sheet. Which is a language used to style a webpage. Our Tree needs to be visually created this is what I would refer to as the DOM aka Document Object Model which is what your browser uses to visually build your page when you load it.

What is meant by "Boxifying Design"?

It is reffering to a technique commonly used as a web developer, as everything on a webpage is basically a box.
When you receive a design mockup you need divide it into boxes to figure out how you will grid and poistion everything on the webpage.

What is the "Box Model"?

All HTML documents are considered boxes. The box model is reffering to the design and layout of the css.
The CSS box model is basically a box that wraps around the page containing margins,borders,paddings and the content of your page.
